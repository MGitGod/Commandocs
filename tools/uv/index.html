    uv¶

    Pythonの高速なパッケージマネージャーである uv のコマンド一覧です。

    基本コマンド¶

    プロジェクトの立ち上げや、基本的な実行に使用するコマンドです。

    プロジェクトの初期化¶

    新しくPythonプロジェクトを作成する際に使用します。

    # 現在のディレクトリに新しいプロジェクトを初期化する
uv init

    使い方: 新しいディレクトリを作成し、その中で実行します。 初期化によって生成される主なファイルは以下の通りです。

    • pyproject.toml
    • プロジェクトの依存関係や設定を管理するファイルです。
    • .python-version
    • 使用するPythonのバージョンが記載されます。
    • README.md

    （補足）生成された pyproject.toml などのテキストファイルを編集する際は、以下のコマンドを使用します。

    # vimを使ってpyproject.tomlを編集する
vim pyproject.toml

    Pythonのインストール¶

    特定のバージョンのPythonをインストールします。

    # Python 3.12をインストールする
uv python install 3.12

    使い方: プロジェクトで指定されたバージョンのPythonがシステムにない場合などに実行します。

    スクリプトの実行¶

    Pythonスクリプトを uv の環境下で実行します。

    # main.pyを実行する
uv run main.py

    使い方: 必要なパッケージがインストールされた隔離環境で、安全にスクリプトを実行できます。

    よく使うコマンド¶

    日々の開発において、パッケージの追加や削除などで頻繁に使用するコマンドです。

    パッケージの追加¶

    プロジェクトに新しいパッケージを追加します。

    # requestsパッケージを追加する
uv add requests

    使い方: pyproject.toml に依存関係が追記され、自動的にインストールとロックファイル（uv.lock）の更新が行われます。

    パッケージの削除¶

    不要になったパッケージをプロジェクトから削除します。

    # requestsパッケージを削除する
uv remove requests

    使い方: 依存関係から削除され、環境からもアンインストールされます。

    環境の同期¶

    pyproject.toml や uv.lock の内容に合わせて、現在の環境を最新の状態に同期します。

    # 依存関係を同期する
uv sync

    使い方: Gitなどで他の人が更新したプロジェクトをプルダウンした直後や、手動で設定ファイルを編集した後に実行します。

    上級者向けコマンド¶

    ツールのグローバルインストールや、プロジェクトのビルドなどを行うコマンドです。

    ツールのインストール¶

    独立した環境にCLIツールをインストールし、どこからでも使えるようにします。

    # ruffをグローバルツールとしてインストールする
uv tool install ruff

    使い方: プロジェクトの依存関係とは別に、LinterやFormatterなどの便利なツールを入れる際に活用します。

    一時的なツールの実行¶

    インストールせずに、一時的にツールをダウンロードして実行します（uvx コマンドと同等の働きをします）。

    # ruffを一時的に実行してチェックする
uv tool run ruff check .

    使い方: 環境を汚さずに、一度だけツールを試したい場合や単発で実行したい場合に使用します。

    プロジェクトのビルド¶

    配布用のパッケージ（Wheelやsdist）をビルドします。

    # パッケージをビルドする
uv build

    使い方: 自作のライブラリをPyPIなどに公開する前の準備として実行します。

    知ってると便利なコマンド¶

    開発をスムーズに進めるための補助的なコマンドです。

    依存関係のツリー表示¶

    現在インストールされているパッケージの依存関係をツリー状に表示します。

    # 依存関係をツリー表示する
uv tree

    使い方: どのパッケージがどのパッケージに依存しているか、競合の原因を調査する際に役立ちます。

    キャッシュのクリア¶

    uv が保持しているダウンロードキャッシュなどを削除します。

    # すべてのキャッシュをクリアする
uv cache clean

    使い方: PCのディスク容量を圧迫している場合や、パッケージのダウンロードに予期せぬ問題が起きた際に実行します。

    仮想環境の明示的な作成¶

    .venv などの仮想環境を明示的に作成します（通常は uv が自動で行うため必須ではありません）。

    # 現在のディレクトリに仮想環境を作成する
uv venv

    使い方: Cursorなどのエディタに仮想環境を確実に認識させるために、あらかじめ手動で作成しておきたい場合に使用します。
