# GitHub

日々の開発で利用する主要なGitおよびGitHubコマンドのまとめです。

## 基本コマンド

バージョン管理を始めるための基本的なコマンドです。

### リポジトリの初期化

新しくGitによる管理を始める際に使用します。

```bash
# 現在のディレクトリをGitリポジトリとして初期化する
git init
```

### リポジトリの複製

リモートリポジトリ（GitHubなど）からローカルにソースコードをダウンロードします。

```bash
# URLを指定してリポジトリをローカルにクローンする
git clone [https://github.com/user/repo.git]
```

## よく使うコマンド

日常の開発フローで最も頻繁に使用するコマンド群です。

### 変更の確認

現在の変更状況やステージングの状態を確認します。

```bash
# 変更されたファイルの一覧や現在のブランチを確認する
git status
```

### ステージングへの追加

変更したファイルをコミットの対象として登録（ステージ）します。

```bash
# 特定のファイルを追加する
git add file.txt

# 変更されたすべてのファイルを追加する
git add .
```

### コミットの作成

ステージされた変更にメッセージを付けて履歴として記録します。

```bash
# メッセージを付けてコミットする
git commit -m "機能追加: ログイン画面の実装"
```

### リモートへの送信

ローカルのコミット履歴をGitHubなどのリモートリポジトリに反映させます。

```bash
# mainブランチに変更をプッシュする
git push origin main
```

### リモートからの取得

リモートリポジトリの最新の変更を取り込み、ローカルブランチに統合します。

```bash
# リモートの最新状態を取得し、現在のブランチにマージする
git pull origin main
```

## 上級者向けコマンド

履歴の改変や複雑な操作を行うためのコマンドです。

### コミット履歴の書き換え

ブランチの起点を変更したり、コミット履歴を整理したりする際に使用します。

```bash
# mainブランチの最新のコミットの後に、現在のブランチの変更を適用し直す
git rebase main
```

### 特定のコミットの取り込み

別のブランチから特定のコミットだけを現在のブランチにコピーして適用します。

```bash
# 指定したコミットハッシュの変更を取り込む
git cherry-pick a1b2c3d
```

### 操作履歴の確認

コミットやブランチの移動など、HEADの変更履歴を確認します。誤った操作を取り消す際に強力です。

```bash
# 過去のすべての操作履歴を表示する
git reflog
```

## 知ってると便利なコマンド

作業を一時中断したり、設定ファイルを編集したりする際の便利なコマンドです。

### 作業の一時退避

コミットしていない変更を一時的に退避させ、作業ディレクトリをクリーンな状態にします。

```bash
# 現在の変更を一時的に保存する
git stash

# 退避した変更を元に戻す
git stash pop
```

### ファイルの編集と除外設定

Gitで管理しないファイルを指定するため、`.gitignore` ファイルを作成・編集する例です。

```bash
# vimを使用して .gitignore ファイルを作成・編集する
vim .gitignore
```

### 変更の取り消し

作業ディレクトリの変更を取り消して、直前のコミットの状態に戻します。

```bash
# 指定したファイルの変更を取り消す
git restore file.txt

# ステージングを解除する（ファイルの内容は保持）
git restore --staged file.txt
```
