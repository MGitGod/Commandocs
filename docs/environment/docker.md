# Docker

Dockerの各種コマンドを目的別にまとめたリファレンスです。

## 基本コマンド

Dockerの動作確認や全体の状態を把握するためのコマンドです。

### バージョンの確認

Dockerが正しくインストールされているか、バージョン情報を確認します。

```bash
# Dockerのバージョンを表示する
docker --version
```

**使い方の例:**
環境構築後や、バージョンに依存する機能を使う前に確認します。

### システム情報の表示

Docker環境の全体的なシステム情報を表示します。

```bash
# コンテナやイメージの数、ストレージドライバなどの情報を表示する
docker info
```

**使い方の例:**
システムのリソース状況や、設定が正しく反映されているかを確認する際に使用します。

## よく使うコマンド

日常的なコンテナの操作やイメージの管理で使用するコマンド群です。

### イメージのダウンロード

Docker Hubなどのレジストリからイメージを取得します。

```bash
# nginxの最新イメージをダウンロードする
docker pull nginx:latest
```

**使い方の例:**
新しくコンテナを立ち上げる前に、必要なベースイメージを用意します。

### イメージのビルド

Dockerfileから新しいDockerイメージを作成します。

```bash
# 現在のディレクトリにあるDockerfileを使用してイメージをビルドする
docker build -t my-app:1.0 .
```

**使い方の例:**
自作のアプリケーションをコンテナ化する際に実行します。
ビルドを行う前に、以下のコマンドでDockerfileを作成・編集します。

```bash
# vimを使用してDockerfileを編集する
vim Dockerfile
```

### コンテナの起動

イメージからコンテナを作成し、起動します。

```bash
# バックグラウンドでnginxコンテナを起動し、ポートをマッピングする
docker run -d -p 8080:80 --name my-nginx nginx
```

**使い方の例:**
Webサーバーなどをローカル環境でテスト起動する際に便利です。

### コンテナの一覧表示

現在稼働中のコンテナ一覧を表示します。

```bash
# 稼働中のコンテナを表示する
docker ps
```

**使い方の例:**
どのポートが使われているか、コンテナのIDは何かを調べたい時に使います。

- 補足オプション
  - `-a` : 停止中のコンテナも含めてすべて表示します。

### コンテナの停止

稼働しているコンテナを安全に停止します。

```bash
# 指定したコンテナを停止する
docker stop my-nginx
```

**使い方の例:**
作業が終わり、不要になったコンテナの動作を止めたい時に使います。

### コンテナの削除

停止しているコンテナを削除します。

```bash
# 指定したコンテナを削除する
docker rm my-nginx
```

**使い方の例:**
不要なコンテナを削除して、ディスク容量を解放する際に使います。

## 上級者向けコマンド

ネットワークの構築やデータ永続化など、より複雑な構成を作るためのコマンドです。

### ネットワークの作成

コンテナ同士を通信させるための独立したネットワークを作成します。

```bash
# 新しいブリッジネットワークを作成する
docker network create my-network
```

**使い方の例:**
Webアプリケーションコンテナとデータベースコンテナを安全に通信させるために使用します。

### ボリュームの作成

コンテナが破棄されてもデータを永続化するためのボリュームを作成します。

```bash
# データを永続化するためのボリュームを作成する
docker volume create my-data
```

**使い方の例:**
データベースのデータや、ユーザーがアップロードしたファイルを保存しておくために利用します。

### Composeによる一括起動

複数のコンテナを一括で構築・起動します。

```bash
# バックグラウンドで複数のコンテナを起動する
docker compose up -d
```

**使い方の例:**
Webサーバー、バックエンド、データベースなどを組み合わせたシステムを一度に立ち上げる際に必須のコマンドです。

## 知ってると便利なコマンド

開発中のデバッグや、トラブルシューティングを効率化するコマンドです。

### コンテナ内でのコマンド実行

稼働中のコンテナ内に入り、コマンドを実行します。

```bash
# 稼働中のコンテナ内でシェルを起動する
docker exec -it my-nginx /bin/bash
```

**使い方の例:**
コンテナ内で設定ファイルを直接確認したり、パッケージを手動でインストールしてテストする際に使います。

### ログの確認

コンテナの標準出力（ログ）を確認します。

```bash
# コンテナのログをリアルタイムで監視する
docker logs -f my-nginx
```

**使い方の例:**
アプリケーションがエラーを出して停止した際など、原因究明のためにログを確認します。

### 不要なリソースの一括削除

使用されていないコンテナ、ネットワーク、イメージなどをまとめて削除します。

```bash
# 未使用のリソースをすべて削除する
docker system prune
```

**使い方の例:**
Dockerがディスク容量を圧迫し始めた時に、環境をクリーンアップするために実行します。

### コンテナとホスト間のファイルコピー

ホストマシンとコンテナの間でファイルやディレクトリをコピーします。

```bash
# コンテナ内の設定ファイルをホストの現在のディレクトリにコピーする
docker cp my-nginx:/etc/nginx/nginx.conf ./
```

**使い方の例:**
コンテナ内で自動生成されたファイルを取り出したり、ローカルで編集したファイルをコンテナ内に送り込む時に大変便利です。
