## Ubuntu でペースト時に `^M` がつく問題

## `^M` とは何か？

`^M` は **キャリッジリターン（CR, `\r`）** という制御文字の画面表示です。

| OS | 改行コード |
| --- | --- |
| Windows | `\r\n`（CR + LF） |
| Linux / Mac | `\n`（LF のみ） |

Windows でコピーしたテキストを Ubuntu にペーストすると、`\r` が余分についてきて `^M` として表示されます。

## 解決方法

### ① ファイルに `^M` が混入している場合

**`sed` で一括除去する方法**（一番手軽）

```bash
sed -i 's/\r//' ファイル名
```

**`tr` コマンドを使う方法**

```bash
tr -d '\r' < 入力ファイル > 出力ファイル
```

**`dos2unix` コマンドを使う方法**（専用ツール）

```bash
# インストール
sudo apt install dos2unix

# 変換
dos2unix ファイル名
```

### ② vim で編集中に `^M` を除去する方法

vim のコマンドモードで以下を入力：

```vim
:%s/\r//g
```

ファイルのフォーマットを明示的に unix に設定する方法：

```vim
:set fileformat=unix
:w
```

### ③ 根本対策：シェルの設定（ターミナルエミュレータ側）

Windows Terminal や VSCode のターミナルから WSL にペーストする際に起きやすいです。

`.bashrc` や `.zshrc` に以下を追記すると改善する場合があります：

```bash
# ブラケットペーストモードを有効化（^M を自動除去）
bind 'set enable-bracketed-paste on'
```

設定を反映するには、以下を実行するかターミナルを再起動します：

```bash
source ~/.bashrc
```
